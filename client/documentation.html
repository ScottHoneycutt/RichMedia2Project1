Countries API Documentation

/getCountryCapital

Request methods: GET or HEAD
Responds with the name of the specified country's capital in JSON format. Requires a "country" query parameter. Country parameter is a case sensitive string.
Possible response codes: 200, 400
Example:

/getCountryCapital?country=Angola

/getCountryNamesByRegion

Request methods: GET or HEAD
Responds with a JSON list of the names of all countries inside the specified region. Requires a "region" query parameter. Region parameter is a case sensitive string.
Possible response codes: 200, 400
Example:

/getCountryNamesByRegion?region=Asia

/getFullCountryData

Request methods: GET or HEAD
Responds with the full JSON data of a specified country. Requires a "country" query parameter. Country parameter is a case sensitive string.
Possible response codes: 200, 400
Example:

/getFullCountryData?country=Angola

/getAllCountryNames

Request methods: GET or HEAD
Responds with a list of all country names in the dataset.
Possible response codes: 200
Example:

/getAllCountryNames

/addCountry

Request method: POST
Adds a new country to the dataset. Requires "name", "capital", and "region" body parameters. All parameters are case sensitive strings. If a new country is added, a response showing the new country data in JSON format is returned. If a country already exists with the same name, updates that country to have the specified capital and region instead of creating a new one.
Possible response codes: 201, 204, 400
Parameter example:

name=newCountry&capital=newCapital&region=newRegion

/removeCountry

Request method: POST
Removes the specified country from the dataset. Requires a "name" body parameter. Name parameter is a case sensitive string.
Possible response codes: 204, 400
Parameter example:

name=Angola
